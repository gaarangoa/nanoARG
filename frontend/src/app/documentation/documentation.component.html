About ARGpore

ARGpore Metagenomics Service

The ARGpore service is an automated pipeline designed specifically for the annotation of Antimicrobial Resistance (AMR) genes from metagenomics samples using the MinION nanopore sequence technology.

How to use ARGpore for data analysis

Register with us

Registration is required to submit your nanopore sequencing data into our platform. Please, note that the registration needs a valid email address. If you don't have an account, just click in the login link placed in the top right of the window.

Check the format of your data

Nanopore devices produce raw reads in the FAST5 format. These files provide a variety of information from your samples. However, ARGpore does not require that information. What you need to submit to the ARGpore service is a FASTA file with your long sequence reads. Therefore, you need to substract the sequences before submiting (1D, 2D, both or the best strand).

To obtain FASTA files from the FAST5 format you can use poretools , a flexible toolkit for exploring datasets generated from the nanopore sequencing devices. It can be as simple as typing: poretools fasta --type best test_data/ to a linux/mac console to get the formated required for ARGpore. For more details about poretools and how to install it please visit their documentation .

How to use ARGpore

Create a New Project

To create a New Project, go under the New project pannel and type the information that is required. Note that the description has to have at least 50 characters. Then click on Create Project button. If everything is right, you will get an update in the dashboard showing the new created project.

Add samples to your project

Once you created your project click on View Project button Then, click on Add Sample button in the project information panel and you will be prompted with a panel with four different steps that you need to go through.

Metadata

Requires information about your sample. Note that blank fields are not allowed.

  • Sample Name: Name of the sample you are analizing.
  • Primary Group: Data related to the sample collection. For instance, soil, water, wastewater, etc.
  • Secondary Group: More specific information about the sample location. For instance, if you primary group is wastewater, the secondary group can be influent, efluent, sludge, etc.
  • Location: The name of the city/town/place where the sample is collected.
  • Comments: Any relevant information related to the sample collection. If there are not relevant comments just put none.

Upload Files

Browse your reads and barcodes files. These files has to be in FASTA format.

Parameters setup

In the current version, there are not any customizable parameter.

Execute

Make sure you are submiting the right files and click on Execute Analysis button.

Retrieving your sample analysis

Samples submitted for analysis will be sent to one of the ARC clusters at Virginia Tech. The time required to run the jobs will vary depending on the cluster availability. In general, it takes from few minutes to few hours to get the analysis done. You will need to monitor the analysis activity under the samples table (see below). It will prompt the status of each sample. If your sample has the status RetrieveResults: SUCCESS, your sample has been processed. If there were any error during the execution, you will see an error message. If the process is going you will see a START message.

Once the samples are processed you can download a table with the absolute abundance of the Antibiotic resistance categories (types) or groups (subtypes) that can be used for downstream analysis.
