Processing nanoARG JSON files in Python¶

In this tutorial you will learn the structure of the JSON file that can be downloaded from nanoARG for each individual sample as well as basic analysis of the data in the file.

In this tutorial you will learn:

  • Identify the SCHEMA of the JSON file
  • How to manipulate JSON files from nanoARG
  • Produce simple visualizations

JSON SCHEMA¶

JSON denotes Java Script Object Notation, a structure-based file that provides a human readable access to the data information.

nanoARG JSON contains all the results from each individual sample. To start, lets import the dependencies. Make sure you have installed the following modules.

Explore the content of the JSON file¶

The nanoARG JSON file consists of an array of five positions. Each position contains a particular information from the sample as follows:

  • data[0]: Contains the reads with at least one ARG
  • data[1]: Contains the nodes and edges needed for the network visualization
  • data[2]: ARGs identified in the sample and colors used in the visualizations (for internal use in nanoARG)
  • data[3]: Counts of each taxa identified in the sample
  • data[4]: Several general data including: total number of reads, total number of mapped ARG reads, others.

In the next sections we will describe in detail each one of the fields.

Reads field (data[0])¶

The reads field is an array of objects, each object has a "reads" and "data" fields. The read field contains all information for the whole read, whereas the data field contains the identified genes in that read. In the example below you can find an example of a the different fields included in each component.

Compare alignment quality bewteen two datasets¶

In this section we are comparing the quality in sequence alignment between two different samples. In this case, we are looking at the effect of correction in the raw reads. This example is useful if you want to compare if two samples are giving you the same alignment quality for ARGs.

In [73]:
# Get only ARGs
def subset(reads, identity=90, coverage=0.9, prob=0.9):
    for read in reads:

        for gene in read['data']:
            if gene['origin'] == 1:
                gene['arg_id'] = gene['metadata'][0]
                gene['arg_class'] = gene['metadata'][3]
                gene['arg'] = gene['metadata'][4]

                del gene['metadata']
                del gene['value']

                if gene['identity']>=identity and gene['coverage']>=coverage and gene['score']>=prob:
                    yield gene


data = json.load(open('che_inf_raw.json'))
reads = data[0]
raw = [ i for i in subset(reads, identity=20)]
raw = pd.DataFrame(raw)
raw['sample'] = 'rawreads'

data = json.load(open('che_inf_canu_corrected.json'))
reads = data[0]
canu = [ i for i in subset(reads, identity=20)]
canu = pd.DataFrame(canu)
canu['sample'] = 'canu-correct'


sx = pd.concat([raw, canu])
sx.index.name = 'index'
sx.to_csv('quality_alignments.csv')

# t test for distributions 
print(stats.ttest_ind(raw['bitscore'],canu['bitscore']))

p = ggplot(sx, aes(x='bitscore', group='sample', fill='sample'))
p + geom_histogram(alpha=0.7, color='black')
p + theme_xkcd()
